3. The context.store API
Every Pages Function receives a context object that includes a persistent key-value store accessible via
context.store. This store provides a simple but powerful interface for maintaining state across requests.
The store supports the following operations: context.store.get(key) retrieves a value by key, returning null if
not found. context.store.put(key, value) writes a value (string or ArrayBuffer) with an optional expiration TTL
in seconds. context.store.delete(key) removes a key-value pair.
Common use cases for context.store include: storing conversation history for chatbots, caching API
responses to reduce latency, maintaining user session data, and tracking rate limits per client IP.
The store is globally distributed and eventually consistent, with writes becoming visible across all edge
locations within seconds. Each key can store up to 25 MB of data, and there is no limit on the number of keys
per namespace.
