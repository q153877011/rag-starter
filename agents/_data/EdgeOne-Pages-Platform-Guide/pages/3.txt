2. Pages Functions Architecture
Pages Functions are serverless functions that run at the edge, close to your users. They are defined by
placing JavaScript or TypeScript files in the /agents directory of your project. Each file automatically maps to
an API route based on its filename.
For example, a file at agents/chat.ts becomes accessible at the /chat API endpoint. The function receives a
request context object that provides access to the request body, headers, query parameters, and a persistent
key-value store.
The execution model is event-driven: each incoming HTTP request triggers a function invocation. Functions
can return standard Response objects, stream data using Server-Sent Events (SSE), or return JSON
payloads directly.
Key architectural features include: (1) automatic route mapping from file structure, (2) built-in KV storage via
context.store for session persistence, (3) environment variable injection for secrets management, and (4)
support for both synchronous and streaming responses.
