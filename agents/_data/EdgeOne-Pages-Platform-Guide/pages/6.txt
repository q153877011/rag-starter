5. Environment Variables and Secrets
EdgeOne Pages provides a secure environment variable system for managing API keys, database
credentials, and other sensitive configuration. Variables are encrypted at rest and injected into the function
runtime at execution time.
Environment variables are configured through the EdgeOne dashboard or CLI. They are accessible in Pages
Functions via the standard process.env object or through the context.env property.
Best practices for environment variable management: (1) Never commit secrets to version control - use
.env.example files with placeholder values. (2) Use descriptive naming conventions like
AI_GATEWAY_API_KEY and AI_GATEWAY_BASE_URL. (3) Rotate credentials regularly and use the
dashboard to update them without redeployment.
For local development, create a .env file in your project root with the same variable names. The EdgeOne CLI
will automatically load these during local testing with the 'eo dev' command.
