4. Streaming Responses with SSE
Pages Functions support Server-Sent Events (SSE) for real-time streaming responses. This is essential for
AI applications where language model outputs are generated token by token and should be displayed
incrementally to the user.
To implement SSE streaming, the function creates a TransformStream, writes events to the writable side, and
returns a Response with the readable side as the body. The Content-Type header must be set to
'text/event-stream'.
Each SSE event follows the format: 'event: <type>\ndata: <json>\n\n'. Common event types in AI applications
include 'text_delta' for incremental text, 'tool_called' for function call notifications, 'citation' for source
references, and 'done' for stream completion.
Error handling in streaming responses requires special attention. If an error occurs mid-stream, the function
should emit an 'error' event with details before closing the stream, allowing the client to display appropriate
error messages.
