三、开发工作流：端到端的  AI 驱动流程
Tony 的开发流程是迄今受访者中最系统化的全  AI 化流程—— 从需求理解到项目规划到并行执行，每个阶段都深度依
赖  AI ，人的角色从 " 写代码的执行者 " 转变为 " 与  AI 协作的项目管理者 " 。
阶段一：需求理解  → 与  AI Brainstorm
拿到需求和  spec 后，第一件事不是动手写代码，而是喂给  Claude Code 一起  brainstorm。反复讨论直到  Tony 对
需求和实现路径都足够清晰，然后让  AI 拆分成带依赖关系的里程碑。这个阶段  Tony 强调 " 花很长时间 "—— 不急于写
代码，而是确保方向正确。
"First thing I do is I feed it into Claude Code — explain to me what this is, and how we can approach implemen
ting it. We start brainstorming until I have a fairly good understanding. That takes quite a while."
「第一件事是把需求喂给  Claude Code—— 解释这是什么、怎么实现。我们一起  brainstorm 直到我充分理解。
这会花不少时间。」
阶段二： AI 创建  Linear Tickets + 设计文档
让  Claude Code 自动在  Linear （项目管理工具）中创建  tickets ，设定依赖关系和  critical path 。 Tony 对  ticket 的要求
很高 ——每个  ticket 必须包含足够的上下文，让任何工程师或  AI agent 都能独立执行。 AI 在理解依赖关系和创建结
构化任务方面表现特别好。同时让  AI 撰写设计文档，经团队  review 后开始执行。
"I put in enough context so that anyone who picks it up — either engineer or a bot — can have enough context
to work on it independently."
「每个  ticket 放足够的  context ，无论是工程师还是  bot 拿到都能独立完成。」
