## Project Overview

An enterprise RAG (Retrieval-Augmented Generation) agent built with React + Vite (frontend) and OpenAI Agents SDK (Python) on EdgeOne Pages (backend). The system processes PDF documents into a local knowledge base and provides a chat interface with citation-backed answers.

## Commands

```bash
# Frontend dev server (Vite)
npm run dev

# Backend agents dev server (EdgeOne Pages)
npm run dev:agents

# Prepare RAG data (extract PDFs → agents/_data/)
npm run prepare-rag
# Or directly:
python3 public/prepare-rag/prepare_rag_data.py

# Build (runs prepare-rag then vite build)
npm run build

# Install Python backend dependencies
pip install -r agents/requirements.txt
```

Python dependency for RAG prep: `pip install pypdf>=4.0.0`

## Architecture

### Two-Layer System

**Frontend** (`src/`): React 19 SPA using Vite. Communicates with backend via SSE streams.
- `src/api.js` — API client handling SSE parsing and stream callbacks (UIMessageStream protocol)
- `src/components/RagChat.jsx` — Main chat UI with streaming, tool visibility, and citation display
- `src/components/KnowledgeBaseSummary.jsx` — Shows knowledge base stats from `/rag-stats`
- `src/components/CitationCard.jsx` — Renders source citations from fetchPages tool output

**Backend** (`agents/`): EdgeOne Pages Python serverless using OpenAI Agents SDK.
- File-based routing: `agents/{path}/index.py` → `POST /{path}`
- Private modules (prefixed with `_`, not routed): `agents/_model.py`, `agents/_loader.py`, etc.
- Data directory: `agents/_data/` (generated by prepare-rag script)

### Backend Modules

| Module | Purpose |
|--------|---------|
| `_agent.py` | Two Agent instances (RAG with tools vs Direct without) |
| `_tools.py` | 5 function_tools: `search_document`, `fetch_pages`, `get_weather`, `get_user_timezone`, `calculate` |
| `_loader.py` | Reads `agents/_data/` filesystem — documents, metadata, page content. Path-traversal-safe. |
| `_model.py` | LLM config via `AsyncOpenAI` + `OpenAIChatCompletionsModel` |
| `_logger.py` | Tagged logger with ISO timestamps |

### API Endpoints

| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/chat/rag` | POST | Streaming RAG chat (uses search_document + fetch_pages tools) |
| `/chat/direct` | POST | Streaming direct chat (no tools) |
| `/stop` | POST | Abort active agent run |
| `/history` | POST | Retrieve conversation history (filters SDK internal items) |
| `/rag-stats` | GET | Knowledge base statistics |

### RAG Data Pipeline

1. Place PDFs in `public/prepare-rag/files/` (optionally with same-name `.json` for tree-structured index)
2. Run `prepare_rag_data.py` → outputs to `agents/_data/`
3. Data layout: `agents/_data/{docId}/meta.json`, `structure.json` (optional), `pages/{n}.txt`
4. At runtime, `_loader.py` reads these files via `Path(__file__).parent / "_data"`; tools use them to serve LLM queries

### Key Conventions

- **Conversation identity**: `pages-agent-conversation-id` header links requests to sessions; `/stop` must NOT include this header (uses body instead)
- **Session memory**: Managed by EdgeOne `context.store.openai_session(conversation_id)`
- **Stream protocol**: Backend emits `data: {JSON}\n\n` SSE events with types: `start`, `text-start`, `text-delta`, `text-end`, `tool-input-available`, `tool-output-available`, `finish`, `error`
- **Agent max turns**: RAG agent allows 6 turns (multi-tool), Direct agent allows 2
- **Handler pattern**: Streaming handlers are async generators yielding SSE strings; non-streaming handlers return dicts

## Environment Variables

Required in `.env` for backend:
- `AI_GATEWAY_API_KEY` — LLM API key
- `AI_GATEWAY_BASE_URL` — LLM API endpoint
- `AI_GATEWAY_MODEL` — Model name

## Important: Do Not Modify

- **`.edgeone/`** — Auto-generated when running `edgeone pages dev`. Never edit files in this directory; modify the project source code instead.
- **`agents/_data/`** — Generated by `prepare_rag_data.py`. Regenerate with `npm run prepare-rag`.
